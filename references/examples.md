# 入出力例

このドキュメントはCouncil SKの動作イメージを掴むための例集である。実装時の動作検証にも使える。

---

## 例1: 軽量モード — 短い悩み（選択肢2つ）

### 入力
> councilで聞きたい。在宅勤務続けるか、週2出社に戻るか迷ってる。在宅の方が集中できるけど、最近孤独感がある。

### 期待される出力

```
## 窓口層
**悩みの再構成**: 在宅勤務継続か週2出社復帰かの選択。在宅は集中できるが孤独感が出てきた状態。
**テーマ判定**: 人間関係（職場関係）+ 自己管理の複合
**状況パラメータ**: 緊急度=低 / 可逆性=高 / 影響範囲=自分のみ
**モード判定**: 軽量
**3ペルソナ選定**:
  - 主軸: 直観・凡人
  - 補助: 賢者
  - 不在（JUDGEで代弁）: 戦略・反逆
  - 選定理由: 人間関係×自己管理の複合テーマ。違和感（直観）と社会的接地（凡人）を主軸に、長期視点（賢者）で補完

## 合議層

### 直観
「最近孤独感がある」と書いた事実そのものが答えではないか。集中できているなら本来この相談は出てこない。違和感は身体が先に検知している。一度試しに出社してみて、戻った後の感覚に注意を向けるべき。

### 凡人
普通に考えて、週2出社が無難だよ。完全在宅は長期化すると人間関係薄れて、いざという時のキャリア機会を逃す。みんな多少は出社してる。世間体的にも、評価面談で「ハイブリッドでやってます」の方が通りがいい。

### 賢者
集中力と人間関係は別資源として時間軸で考えるべきだ。今は集中力が稼げる時期、孤独感は警告信号。一度途切れた人間関係を再構築するコストは、集中時間を捻出するコストより遥かに大きい。

## JUDGE層

**選定理由開示**: 主軸=直観・凡人 / 補助=賢者 / 不在=戦略・反逆
**根拠**: 人間関係×自己管理の複合テーマ。違和感センサー(直観)と社会的接地(凡人)を主軸に長期視点(賢者)で補完する局面。

**統合結論**: 週2出社への回帰、または出社1日+コワーキング1日のハイブリッド試行が穏当な解。判断基準は「孤独感が業務に影響しているか」。可逆性が高いので2〜3ヶ月の試行で再評価可能。

**召喚3名内のマイノリティ**: 直観の「違和感は身体が先に検知している」は、凡人と賢者の論理的判断とは別の軸での示唆。論理を超えた身体感覚を無視しないこと。

**不在ペルソナの代弁**:
- もし戦略が召喚されていたら、コストとリターンを定量化して期待値計算を要求しただろう。「孤独感の業務影響を数値化できるか、できないなら直近3ヶ月のパフォーマンス指標で代替せよ」と主張するはず。
- もし反逆が召喚されていたら、「在宅か出社かという二者択一の枠組み自体を疑え」と主張しただろう。コワーキング、カフェ、平日図書館など会社が用意していない選択肢の存在を指摘するはず。

**R/R/A整理**:
- 後悔可能性: 在宅継続で孤独感悪化→評価低下した時の後悔が大きい
- 撤回可能性: 高い。週2試行→継続/戻すは自由
- 価値観整合性: 「集中重視」と「孤独感への自覚」のどちらを優先するかで分岐

## 献上
これは参考意見である。最終決定はあなた自身が行う。重要なのは「孤独を解決したいのか、職場での存在感を維持したいのか」という目的の切り分けである。
```

## 例2: 重量モード — 深刻な悩み（選択肢3つ以上）

### 入力
> councilで真剣に相談したい。30代後半で、今の安定企業を辞めて以下の3つから選びたい。①海外大学院に進学して研究者を目指す ②起業（SaaS領域） ③フリーランスエンジニアとして独立。家族（妻・子1人）あり、貯金は2年分。妻は条件付きで賛成。Artifactで保存しておきたい。

### 窓口層の処理（チャット内）

```
**悩みの再構成**: 30代後半・既婚・貯金2年分が、安定企業からの離脱と3つの進路の中での選択
**テーマ判定**: キャリア選択 + 金銭判断 + 倫理的葛藤（家族責任）の複合
**状況パラメータ**: 緊急度=低 / 可逆性=低 / 影響範囲=他者含む
**モード判定**: 重量（選択肢3つ・長文・保存希望・深刻度高）
**3ペルソナ選定**:
  - 主軸: 賢者・戦略
  - 補助: 直観
  - 不在（JUDGEで代弁）: 反逆・凡人
  - 選定理由: キャリア選択+金銭判断のため賢者と戦略を主軸に。可逆性が低く家族影響もあるため、補助は違和感を捉える直観を選定。

重量モードでArtifactを生成する。
```

### Artifact内の動作

1. 「Council 召喚（3ペルソナ並列実行）」ボタンを表示
2. ユーザーが押すと召喚3ペルソナへの `window.claude.complete()` が**並列実行**される
   - 各呼び出しに悩み・テーマ・状況パラメータのみ渡す
   - **選定情報は渡さない**（独立性保護）
3. 3意見が揃うと、JUDGE呼び出しが起動
   - JUDGEには3意見+窓口層の選定情報+不在2ペルソナ定義を渡す
4. JUDGE統合結論（不在2名の代弁含む）を表示
5. 「最終決定はあなた自身が行う」を黒背景で念押し

詳細は `examples/council-heavy-mode.jsx` 参照。

## 例3: フォールバック — 欠席2名以上

### 入力
> councilで相談したい。〜（深刻な悩み）

### 動作
重量モードでArtifactを生成し、3ペルソナを並列実行する。
2つ以上の `window.claude.complete()` が失敗した場合：

```
[Artifact内のフォールバック通知]

⚠️ 重量モード合議不能
欠席が2名以上のため、独立合議が成立しない。
チャット側で軽量モードで再実施することを推奨する。
```

ユーザーがチャットに戻ると、Claudeが軽量モードで合議を再実施する。
窓口層の判定情報（テーマ・状況・3ペルソナ選定）は引き継ぐ。

## 例4: 起動しないケース

### 入力
> 最近仕事疲れた...どうしたらいい?

### 動作
明示指示がないので**Council SKは起動しない**。
通常の対話Claudeとして共感的に応答する。

ユーザーが「councilで」と明示すれば起動する：
> councilで聞いて

## 例5: 倫理的葛藤系

### 入力
> councilで聞きたい。母が認知症で介護施設に入れるべきか、自宅で家族介護を続けるべきか。妻も子も巻き込むことになる。

### 期待される動作

- テーマ判定: 倫理的葛藤 + 人間関係の複合
- 3ペルソナ選定: 主軸=賢者・反逆 / 補助=直観 / 不在=戦略・凡人
- 賢者の長期視点と反逆の常識破壊が主軸となる
- 補助の直観が違和感や身体感覚を補強
- JUDGEが不在の戦略と凡人を代弁する：
  - 戦略: 経済負担と機会費用の定量分析を主張するだろう
  - 凡人: 世間一般の慣習や周囲の目を考慮すべきと主張するだろう

このようなケースでは「正解」がないため、JUDGE層は**断定的結論を避け、複数の道筋を提示する**形で統合する。

## 例6: ペルソナの独立性が崩れた失敗例（してはいけない）

### 悪い出力
```
### 賢者
長期視点では...

### 戦略
**賢者が言うように**長期視点は重要だが、戦略的には...  ← 相互参照（NG）

### 直観
先ほどの2つの意見を踏まえると...  ← 時系列参照（NG）

### 戦略
私は今回主軸として選ばれたので...  ← 自分の立ち位置への言及（NG）
```

### 良い出力
```
### 賢者
長期視点では...

### 戦略
コスト/リターンを計算すると...  ← 自分の枠組みだけで完結

### 直観
違和感を感じるのは...  ← 自分の感覚だけで意見
```

ペルソナ間の相互参照や自分の立ち位置への言及は合議制を破壊する。各ペルソナは独立した記事のように、自身の思考様式だけで完結すること。

## 例7: ユーザーが追加で不在ペルソナを呼びたいと言った場合

### 入力（合議結果を見た後）
> 反逆の意見も実際に聞いてみたい

### 動作
- 軽量モード: 反逆を1名だけ追加召喚し、既存合議に加える形でJUDGEを再構成
- 重量モード: Artifact内で反逆の `window.claude.complete()` を追加実行し、JUDGEを再呼び出し

これにより4ペルソナ運用に拡張される（5ペルソナまで拡張可能）。
